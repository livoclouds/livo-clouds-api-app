dashboard.service.ts:136-146 (the summaries.length === 0 fallback) references condominium_id, transaction_date, and flow_type in snake_case, but the actual Postgres columns are camelCase identifiers ("condominiumId", "transactionDate", "flowType"). This fallback will throw "column does not exist" if exercised. Out of scope for Phase 1 (the roadmap targeted only the Set→SQL conversion). Path is rarely hit because FinancialMonthlySummary rows usually exist for any year with transactions. Recommend a follow-up correctness fix.
R4.2 unmitigated

Petty-cash parallel reads not bundled
The perf-only suggestion to wrap findFirst + count in Promise.all was scoped out of Phase 1. Documented as a follow-up.

Trim id fields from inner selects (deferred)
